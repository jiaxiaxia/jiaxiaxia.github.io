javascript中值类型与引用类型

要说值类型与引用类型，就得先说一下操作系统中的栈与堆

操作系统中的栈：

  • 由编译器自动分配和自动释放，一个函数对应一个栈，用于存放函数的参数值、函数调用完成后
    的返回值和函数体内的局部变量等。栈占用连续的一段内存空间，其操作和组织方式与数据结构中
    的栈十分相似。栈是为了执行线程留出的内存空间。当调用函数时创建栈，当函数执行完毕，
    栈就被回收了。

操作系统中的堆：

  • 由程序员手动进行内存的申请与释放，由于程序员手动申请及释放的内存块存放在堆中，堆
    中有很多内存块，所以堆的组织方式类似于链表。操作系统中的堆与数据结构中的堆完全不同。
    堆是为动态分配预留的内存空间，其生命周期为整个应用程序的生命周期。当应用程序结束以后，
    堆开始被回收。

  • 每个线程都有一个属于自己的栈，但每个应用程序通常只有一个堆（也有多个堆的情况）。
    当线程被创建的时候，设置了栈的大小。在应用程序启动的时候，设置了堆的大小。栈的大小通常是
    固定的，但是堆可以在需要的时候进行扩展，如程序员向操作系统申请更多内存的时候。

  • 由于栈的工作方式类似于数据结构中的栈，堆的工作方式类似于链表，所以栈显然会比堆快多。
    按照栈的存取方式，想要释放内存或新增内存，只需要相应移动栈顶指针即可。堆则要首先在内存的空闲区域寻找合适的内存空间，然后占用，然后指向这块空间。

  • 好了，关于堆与栈的介绍，就此打住，不然该篇文章就是关于堆栈的讨论了。

在ECMAScript中，变量可以存在两种类型的值，即原始值与引用值

原始值

存储在栈中的简单数据段，也就是说，它们的值直接存储在变量访问的位置

引用值

存储在堆中的对象，也就是说，存储的变量处（栈）的值是一个指针，指向存储对象的内存冲（堆）

原始值与引用值的复制（变量的赋值）：

  • 原始值复制时，计算机会为新的变量在栈中申请区域，然后将变量（旧）的值复制一份赋值给新的变量

  • 引用值型复制时，同样的计算机会为新的变量在栈中申请区域，并将变量（旧）的值（存放真实值的地址）复制一份复制给新的变量，注意，这里存在于堆中的值并不会复制，只是在栈中多出了一个指向堆中相同地址的变量。具体如下图所示：

那么javascript中函数参数的传递又是怎样的呢

其实在讲参数的传递之前，首先得了解下函数参数的几种传递类型

值传递：计算机会为形参申请新的地址，然后将实参的值完全拷贝一份，复制给形参。这样，形参和实参在计算机是两个相互独立的内存。彼此不会影响。

引用传递：形参的值保存的是实参所在内存地址。所以当你在函数内部操作形参时，其实是在操作实参的值

javascript中函数参数无论是原始值值类型还是引用类型，参数是值传递类型的，下面具体说一下既然是按照值传递，那么根据值传递的定义来说的话：不管参数是原始值类型的，还是引用类型的。都会为形参申请新的地址，并将实参的值，值，值，不是地址复制给形参。需要注意的是，当参数类型为引用类型是。形参的值其实是与实参指向堆中同一地址的值。所以就会发生，你在操作形参时，其实是在操作堆中的地址。导致实参的值也会发生边。除非，你给新参重新赋值，不在与实参指向同一个地址。
